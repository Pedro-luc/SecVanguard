File Upload

Uma vulnerabilidade crítica de injeção de arquivos maliciosos

O que é uma Falha de File Upload?

Uma falha de File Upload é uma vulnerabilidade web em que um atacante consegue fazer o upload de arquivos maliciosos para o lado do servidor, por isso o ataque é um ataque server-side que ocorre do lado do servidor.

Esta vulnerabilidade surge geralmente em formulários que não fazem um filtro sanitário impedindo que outros formatos de arquivos duvidosos sejam enviados e por este motivo esta vulnerabilidade é explorada dentro de formulários vulneráveis.

Impactos causados por esta vulnerabilidade

  • RCE (Remote Code Execution): Execução remota de códigos, permitindo o atacante controlar o sistema comprometido.
  • Deface: Ataques onde o modelo original da página é alterado, causando uma deformidade do conteúdo original.
  • Roubo de informações: O atacante, através de uma backdoor, consegue se infiltrar no sistema para colher informações.
  • Ataques de negação de serviço (DoS): O envio de arquivos em grandes quantidades pode sobrecarregar o sistema, tornando-o indisponível.

Exemplos de CVEs Relacionadas

  • CVE-2018-9206: A vulnerabilidade afetava o CMS Chamilo e-learning, onde o sistema não verificava o tipo de arquivo enviado, permitindo que um invasor carregue arquivos executáveis, resultando em uma execução remota de códigos.
    Impacto CVSS: 9.8 (Crítico)
  • CVE-2020-3452: Envolvia o Cisco Adaptive Security Appliance (ASA) e Cisco Firepower Threat Defense (FTD). Essa falha permitia que um invasor enviasse arquivos não autorizados, levando à exposição de arquivos confidenciais no sistema comprometido.
    Impacto CVSS: 7.5 (Alta)
  • CVE-2021-22204: A falha afetou o ExifTool, uma biblioteca usada para manipulação de metadados em imagens. Permitindo execução de código remoto (RCE) quando arquivos de imagem maliciosos eram enviados, explorando o file upload em softwares que utilizam essa biblioteca.
    Impacto CVSS: 7.8 (Alta)
