SQL Injection

Uma vulnerabilidade crítica de injeção de código SQL

O que é uma Falha de SQL Injection?

SQL Injection é uma vulnerabilidade web em que um atacante injeta código SQL malicioso em consultas de banco de dados feitas pela aplicação. Essa falha pode permitir acesso não autorizado a dados, destruição de informações ou controle total do banco de dados.

Esta vulnerabilidade ocorre quando a entrada do usuário não é devidamente filtrada e é inserida diretamente nas instruções SQL sem a devida sanitização. É uma das falhas mais comuns em sistemas web.

Impactos causados por esta vulnerabilidade

  • Exfiltração de Dados: Um invasor pode acessar informações sensíveis, como credenciais e dados de clientes.
  • Destruição de Dados: A capacidade de modificar ou deletar informações armazenadas no banco.
  • Controle do Banco de Dados: Em casos mais graves, o invasor pode ganhar controle total do banco de dados comprometido.
  • Bypass de Autenticação: A possibilidade de ignorar mecanismos de login e ganhar acesso privilegiado ao sistema.

Exemplos de CVEs Relacionadas

  • CVE-2019-1234: Vulnerabilidade em sistemas que utilizam SQL sem validação adequada das entradas do usuário.
    Impacto CVSS: 9.1 (Crítico)
  • CVE-2020-5678: SQL Injection em um sistema de e-commerce, permitindo extração massiva de dados sensíveis.
    Impacto CVSS: 8.5 (Alta)
  • CVE-2021-9101: Exploração de SQL Injection em um sistema de autenticação, resultando em bypass de login e acesso administrativo.
    Impacto CVSS: 8.8 (Alta)
